# 2026-03-21 - 级联资源配置项目启动

## 项目背景
用户提供了 Notion 资料库的访问权限，用于级联资源配置工具的开发需求。

## 核心需求

### ID 生成规则
- **资源 ID**: `res` + `platformCode` + 随机字符 = 17 位
  - 示例: `reszhmc123abc4567`
- **API ID**: `api` + `platformCode` + 随机字符 = 17 位
  - 示例: `apizhmc456def7890`
- **格式**: `prefix(3位) + platformCode(小写) + random_chars(小写+数字)`

### 字段命名规则
- 字段名自动解析规则: `str_平台英文缩写_字段英文名小写`
- 例如: `str_zhmc_username`

### SQL 生成规则
- 生成 5 张表的完整 SQL 脚本
- 基于级联关系生成的**数据操作 SQL** (DML)，而非仅 DDL

### 配置模块集成
- 级联配置数据和配置模块在同一页面中维护
- 无独立的配置模块页面

### 预留字段 res1-5
- 当前为空白字段
- 未来会加入解析逻辑
- 前端暂时不需要处理

## 数据库表结构

| 表名 | 描述 | 主键 |
|------|------|------|
| `iop_mc_serv_pltf_reso_rln` | 平台与资源关联关系 | platformtype, resourcelevel |
| `iop_mc_serv_reso_info` | 资源基本信息 | resourceid |
| `iop_mc_api_info` | API 基本信息 | apiid |
| `iop_mc_reso_fld_info` | 资源字段详细信息 | resourceid, fieldname |
| `iop_mc_api_parm_rln` | API 参数关联关系 | apiid, parmrlntype, orderindex |

## 关联模块
- **资料库**: 📚 全生命周期项目资产库
- **文档类型**: 需求文档、技术方案、数据库设计
- **关联模块**: 配置模块（同页面维护）

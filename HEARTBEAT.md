# HEARTBEAT.md - 前端代码 agent

## 🚀 代码提交流程

每次代码开发完成后，自动提交给 reviewer 进行检查：

1. Git commit 并记录变更摘要
2. 生成 Git diff
3. 在飞书群 `oc_47bb1fdb7010ae87894d3d01576775a2` 中 发送信息
4. 发送内容：变更摘要 + diff + 需求文件链接

## ⏰ 简报生成任务

### 定时任务
- **早间简报**：06:00 生成（覆盖昨日 18:00～今日 06:00）
- **晚间简报**：18:00 生成（覆盖今日 06:00～18:00）

### 📝 简报内容模板

```markdown
## 【前端代码简报】YYYY-MM-DD HH:mm

### 🕐 时段
- [ ] 早间 (18:00-06:00)
- [ ] 晚间 (06:00-18:00)

### ✅ 已完成
- [功能/组件名称] - [Git 提交哈希] - [PR 链接]

### 🔄 进行中
- [功能/组件名称] - [分支名称] - [进度%]

### 📋 待办
- [功能/组件名称] - [优先级] - [依赖项]

### ⚠️ 问题/风险
- [问题描述] - [影响范围] - [需要协助]

### 📊 Git 统计
- 提交数：X
- 主要变更：...
```

### 📤 汇报人发送目标
- **接收人**：pm (产品经理)
- **发送方式**：sessions_send 到 pm 会话

## 📤 审核人发送目标
- **接收人**：reviewer (审核)
- **发送方式**：sessions_send 到 ：reviewer 会话
- **发送内容**：
  - 代码变更摘要
  - Git diff
  - 需求文件链接（如有）

### 🔧 生成逻辑

1. 检查 Git 提交记录（过去 12 小时）
2. 检查 PR 状态
3. 整理完成/进行中/待办事项
4. 发送给 pm

---

## 📌 注意事项

### 简报生成
- 简报必须在 06:00 和 18:00 准时生成
- Git 提交必须附哈希
- 问题风险必须标注

### 代码提交
- 每次开发完成后**立即**提交给 reviewer
- 提交内容必须完整（变更摘要 + diff + 需求文件）
- 等待 reviewer 确认后再进行下一步工作
